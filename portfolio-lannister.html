Transaction Payment Splitting Service (TPSS)

LannisterPay has reached out to you to help implement a Transaction Payment Splitting Service (TPSS). The service is meant to calculate the amount due to one or more split payment "entities" as well as the amount left after all splits have been computed.

Github Link

The details of this project centers around me providing an API service which expose a single HTTP POST endpoint split-payments/compute that accepts a transaction object with the following properties:

  • ID Unique numeric ID of the transaction
  • Amount Amount to be splitted between the split entities defined in the SplitInfo array
  • Currency The currency of the transaction
  • CustomerEmail Email address of the transaction customer
  • SplitInfo An array of split entity objects.
